\hypertarget{md_public_2_muse_2assets_2vendor_2bootstrap-icons_2_r_e_a_d_m_e_autotoc_md48}{}\doxysubsection{\texorpdfstring{Usage}{Usage}}\label{md_public_2_muse_2assets_2vendor_2bootstrap-icons_2_r_e_a_d_m_e_autotoc_md48}
Depending on your setup, you can include Bootstrap Icons in a handful of ways.


\begin{DoxyItemize}
\item Copy-\/paste SVGs as embedded HTML
\item Reference via {\ttfamily \texorpdfstring{$<$}{<}img\texorpdfstring{$>$}{>}} element
\item Use the SVG sprite
\item Include via CSS
\end{DoxyItemize}

\hypertarget{md_public_2_muse_2assets_2vendor_2bootstrap-icons_2_r_e_a_d_m_e_autotoc_md51}{}\doxysubsection{\texorpdfstring{Adding SVGs}{Adding SVGs}}\label{md_public_2_muse_2assets_2vendor_2bootstrap-icons_2_r_e_a_d_m_e_autotoc_md51}
Once a new SVG icon has been added to the {\ttfamily icons} directory, you\textquotesingle{}ll need to optimize them. An npm script is used to\+:


\begin{DoxyEnumerate}
\item Optimize our SVGs with SVGO.
\item Modify the SVGs source HTML, removing all attributes before setting new attributes and values in our preferred order.
\end{DoxyEnumerate}

Use {\ttfamily npm run icons} to run the script, run {\ttfamily npm run pages} to build permalink pages, complete those pages, and, finally, commit the results in a new branch for updating.\hypertarget{md_public_2_muse_2assets_2vendor_2bootstrap-icons_2_r_e_a_d_m_e_autotoc_md52}{}\doxysubsection{\texorpdfstring{Publishing}{Publishing}}\label{md_public_2_muse_2assets_2vendor_2bootstrap-icons_2_r_e_a_d_m_e_autotoc_md52}
