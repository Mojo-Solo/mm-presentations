Progress bars are just regular SVG paths. Read \href{http://jakearchibald.com/2013/animated-line-drawing-svg/}{\texttt{ Jake Archibald\textquotesingle{}s blog post}} to see how the path drawing works under the hood.

{\itshape Progress\+Bar.\+js} uses \href{https://jeremyckahn.github.io/shifty/}{\texttt{ shifty}} tweening library to animate path drawing. So in other words, animation is done with Java\+Script using \href{https://developer.mozilla.org/en-US/docs/Web/API/window.requestAnimationFrame}{\texttt{ request\+Animation\+Frame}}. Animating with JS gives more control over the animation and is supported across major browsers. For example IE \href{https://connect.microsoft.com/IE/feedbackdetail/view/920928/ie-11-css-transition-property-not-working-for-svg-elements}{\texttt{ does not support}} animating SVG properties with CSS transitions.\hypertarget{md_public_2_muse_2assets_2vendor_2progressbar_2docs_2index_autotoc_md205}{}\doxysubsection{\texorpdfstring{\{migrations\} Migration guide}{\{migrations\} Migration guide}}\label{md_public_2_muse_2assets_2vendor_2progressbar_2docs_2index_autotoc_md205}
To upgrade from version x to y\+:


\begin{DoxyItemize}
\item Go to \href{https://github.com/kimmobrunfeldt/progressbar.js/releases}{\texttt{ releases page}}
\item Find the version x you are using
\item Browse through release notes from x to y. API breaking releases are marked with {\ttfamily \#breaking}.
\item Fix your current code to use new API
\end{DoxyItemize}
