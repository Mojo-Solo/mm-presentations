Small library to animate elements on your page as you scroll.

You may say it\textquotesingle{}s like WOWJS, yeah -\/ you\textquotesingle{}re right, effect is similar to WOWJS, but I had a different idea how to make such a plugin, so here it is. CSS3 driven scroll animation library.

AOS allows you to animate elements as you scroll down, and up. If you scroll back to top, elements will animate to it\textquotesingle{}s previous state and are ready to animate again if you scroll down.

üëâ To get a better understanding how this actually works, I encourage you to check \href{https://css-tricks.com/aos-css-driven-scroll-animation-library/}{\texttt{ my post on CSS-\/tricks}}.

\hypertarget{md_public_2_muse_2assets_2vendor_2aos_2_r_e_a_d_m_e_autotoc_md35}{}\doxysubsubsubsection{\texorpdfstring{API}{API}}\label{md_public_2_muse_2assets_2vendor_2aos_2_r_e_a_d_m_e_autotoc_md35}
AOS object is exposed as a global variable, for now there are three methods available\+:


\begin{DoxyItemize}
\item {\ttfamily init} -\/ initialize AOS
\item {\ttfamily refresh} -\/ recalculate all offsets and positions of elements (called on window resize)
\item {\ttfamily refresh\+Hard} -\/ reinit array with AOS elements and trigger {\ttfamily refresh} (called on DOM changes that are related to {\ttfamily aos} elements)
\end{DoxyItemize}

Example execution\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{AOS.refresh();}

\end{DoxyCode}


By default AOS is watching for DOM changes and if there are any new elements loaded asynchronously or when something is removed from DOM it calls {\ttfamily refresh\+Hard} automatically. In browsers that don\textquotesingle{}t support {\ttfamily Mutation\+Observer} like IE you might need to call {\ttfamily AOS.\+refresh\+Hard()} by yourself.

{\ttfamily refresh} method is called on window resize and so on, as it doesn\textquotesingle{}t require to build new store with AOS elements and should be as light as possible.\hypertarget{md_public_2_muse_2assets_2vendor_2aos_2_r_e_a_d_m_e_autotoc_md36}{}\doxysubsubsection{\texorpdfstring{Global settings}{Global settings}}\label{md_public_2_muse_2assets_2vendor_2aos_2_r_e_a_d_m_e_autotoc_md36}
If you don\textquotesingle{}t want to change setting for each element separately, you can change it globally.

\begin{DoxyCode}{0}

\DoxyCodeLine{\ \ AOS.init(\{}

\DoxyCodeLine{\ \ \});}

There are several options that you can use to fit AOS perfectly into your project, you can pass one of three device types\+: {\ttfamily mobile} (phones and tablets), {\ttfamily phone} or {\ttfamily tablet}. This will disable AOS on those certains devices. But if you want make your own condition, simple type your statement instead of device type name\+:


\begin{DoxyCode}{0}

\end{DoxyCode}


There is also posibility to pass a {\ttfamily function}, which should at the end return {\ttfamily true} or {\ttfamily false}\+:


\begin{DoxyCode}{0}
